# ROADMAP

## Quick wins (1-2 weeks)
- Add contributor onboarding docs and local validation steps.
- Improve README with development workflow and release checklist.
- Add script aliases for repeatable quality checks (`lint`, `format`, `format:check`).
- Validate all theme JSON files against consistent formatting.

## Medium improvements (2-6 weeks)
- Add screenshot automation for each theme variant.
- Add visual regression checks to detect accidental token changes.
- Introduce semantic-release or changesets for predictable versioning.
- Document color token philosophy (contrast, accessibility, and tone intent).

## Big bets (1-2 quarters)
- Build a theme token generator to produce variants from a single source.
- Add CI pipelines for linting, formatting, and package validation.
- Expand marketplace positioning with curated language-specific previews.

## Strategic rewrites (as needed)
- Migrate from static theme JSON maintenance to generated themes from a typed config schema.
- Create a dedicated docs site for palettes, previews, and changelog history.
